# GitPage로 이주한 이유

Velog를 사용하면서 가장 불편했던 점은 태그 관리였다.

`cpp`, `unreal`, `til`과 같이 태그를 분류해도 일관되게 정리되지 않았고, 글이 늘어날수록 원하는 내용을 찾기가 어려워졌다.

처음에는 세부 분류를 위해 `cpp/vector`, `unreal/bp`, `unreal/actor` 같은 형태로 태그를 추가해 보았지만, 오히려 태그 종류만 늘어나면서 관리가 더욱 복잡해졌다.

내가 원하는 것은 단순히 다른 사람들이 보기 좋은 기술 블로그가 아니다.

필요한 순간에 과거의 기록을 빠르게 찾아보고, 문제를 해결했던 과정과 생각을 다시 확인할 수 있는 **개인 개발 기록 저장소**에 가깝다.

그래서 조금 더 자유롭게 구조를 설계하고 커스터마이징할 수 있는 GitHub Pages(GitPage)로 이주하게 되었다.

## 내가 원하는 개발 기록

개발을 하다 보면 한 번 해결한 문제를 몇 달 뒤에 다시 만나게 되는 경우가 많다.

그때마다 검색을 처음부터 다시 하기보다는, 과거의 내가 어떤 방식으로 해결했는지 빠르게 찾아볼 수 있는 공간이 필요했다.

그래서 앞으로 이 공간은 단순한 학습 정리뿐만 아니라,

* 개발 중 겪었던 문제
* 해결 과정
* 시행착오
* 기술 정리
* 프로젝트 회고

등을 기록하는 용도로 활용할 예정이다.

## 앞으로

GitPage를 사용하면서 겪게 되는 불편함이나 개선하고 싶은 부분들도 이곳에 함께 기록해 보려고 한다.

완벽한 환경을 찾기보다는, 직접 사용하면서 필요한 기능을 추가하고 구조를 다듬어 나가는 과정 자체도 하나의 개발 경험이라고 생각한다.
